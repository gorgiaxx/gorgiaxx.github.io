Introduction

I often use a 4G hotspot when traveling for work. This 4G router is incredibly handy: it supports LTE/UMTS/GSM, can act as a power bank, a router, and a cellular modem, and supports both wireless and wired connections. With a proper unlock, you can customize the IMEI, bypass ICCIDâ€“IMEI binding restrictions, and evade network censorship.

The first thing I did after getting it was to open it up and look for a UART port. I was able to print logs, but it turned out to be password-protected with an 8-character password, and there was no way to recover it by reading NVRAM.

After flashing usbloader, you can use balong_flash to flash arbitrary firmware. However, NVRAM will be wiped, which breaks mobile data connectivity, so you need to restore NVRAM:

mount /dev/block/mmcblk0p1 /sdcard
cd /sdcard
for i in 3 4 5 6 7 23; do cat mtdblock$i > /dev/block/mtdblock$i; done

Then you can get a shell via ADB and telnet. The default username/password is root/changemerightnow.

OLED Hijacking

ValdikSS wrote a tool to modify the OLED menu. The idea is to patch the original OLED programâ€™s shared library: hijack sprintf to set a custom menu, hijack register_notify_handler to handle button events, and redirect them to a specified script. Based on that, we can add our own custom features.

IMEI Binding Bypass

First, refer to my earlier post eSIM Learning Notes which mentions a wire-soldering attack against eSIM.

Before soldering wires, check the original IMEI:

E5885Ls-93aroot@p722:/ # imei
/system/bin/imei [VALUE]
Current IMEI: 358731070934433

With ValdikSSâ€™s firmware, the IMEI is changed on every boot. You can also manually set it to a specific IMEI. You can use existing tools to set it to the target device IMEI; it takes effect after reboot. Switching networks, switching modes, etc. are also done via AT commands.

imei 864758031772807
or
echo -e "AT^CIMEI=864758031772807" > /dev/appvcom

Backup Partition

Backing up is a good habit. Insert a microSD card first, mount it, and then dump NAND partitions to the SD card:

/system/busybox sh
mount /dev/block/mmcblk0p1 /sdcard
cd /sdcard

mkdir mtdblocks
cd mtdblocks
for i in `seq 0 27`; do cat /dev/block/mtdblock$i > mtdblock$i; done

cd ..
mkdir nanddump
cd nanddump
for i in `seq 0 27`; do nanddump -f mtd$i /dev/mtd/mtd$i; done
for i in `seq 0 27`; do nanddump -o -f mtdoob$i /dev/mtd/mtd$i; done

cd ..
tar cf files.tar /system /app /data /root /modem_log /modem_fw /online /mnvm2:0
cat /proc/mtd > procmtd
cat /proc/kallsyms > kallsyms
mount > mount

cd /
umount /sdcard
