汽车总线知识扫盲 - 序

Jun 30 2021

UDS诊断

请求格式

Service ID 服务ID
Sub-Function Byte 子功能字节
Response Code 响应码

A_PDU, Application Protocol Data Unit Specification

应用层协议数据单元直接由A_SDU和A_PCI构成

A_PCI, Application Protocol Control Information

A_PCI, Application Protocol control information, 应用协议控制信息，参数字段总共有两种格式

服务请求和服务响应格式: SI
消极响应: NR_SI, SI

SI, Service Identifier

SID, 服务ID。范围0x00至0xFF
其中有ISO 14229-1定义的ID、预留ID、系统供应商设置的ID。

NR_SI, Negative Response Service Identifier

消极响应ID

NRC_, Supported negative response codes

消极响应码

请求

不带功能ID的请求
不带功能ID的积极响应

服务ID + 子功能字节

UUDT-Response

数据参数

消极响应

消极响应ID + 请求服务ID +　响应码

DiagnosticSessionControl (0x10) service

诊断会话控制

ECUReset (0x11) service

0x01 hardReset，硬重置，重启电源
0x02 keyOffOnReset， 点火开关重置
0x03 softReset, 软重置，重启软件
0x04 enableRapidPowerShutDown，启用快速电源关闭
0x05 disableRapidPowerShutDown，禁用快速电源关闭

0x06-0x3F ISOSAEReserved SAE预留
0x40-0x5F 车厂预留
0x60-0x7E 系统供应商预留
0x7F 预留

因为UDS在应用层实现，所以使用各类总线协议，以下是ISO 14229每一部分的主题

ISO 14229-1 规格和需求
ISO 14229-2 会话层服务
ISO 14229-3 UDS在CAN上的实现(UDSonCAN)
ISO 14229-4 UDS在FlexRay上的实现(UDSonFR)
ISO 14229-5 UDS在以太网协议上的实现(UDSonIP)
ISO 14229-6 UDS在K-line上的实现(UDSonK-Line)
ISO 14229-7 UDS在LIN上的实现(UDSonLIN)

PDU

协议数据单元(PDU)格式如下，前三个是必选参数：

  • Mtype, 消息类型，1字节
  • SA, 源地址，2字节
  • TA, 目标地址，2字节
  • TA_Type, 目标地址类型，1字节
  • Result, 消息传输结果(ok,error)，1字节
  • Length, 数据长度，4字节
  • Data, 数据比特流
